#### Anti-Patterns Verdict

**Does not look AI-generated.** The warm-black + amber palette is specific and editorial. Font pairing (DM Sans + JetBrains Mono) and restrained use of color signal intentional craft. The layout doesn't match any common template.

**Deterministic scan**: Unavailable. Bundled detector dependency (`detect-antipatterns.mjs`) not found in this installation. Manual review substituted.

**Visual overlays**: Not available. No browser automation tooling in this environment.

#### What's Working

1. **Section grid layout** — 1→2→4 column responsive grid is simple and effective. Each section is self-contained with header, total, items list, and add button. Clear information architecture.

2. **Available amount as hero** — The 24px amber number in the fixed footer is the correct focal point. The number animation + glow flash on update gives live feedback without extra UI.

3. **Undo flow with timer bar** — The 6-second undo toast with the shrinking amber timer bar is excellent. Functional delight: communicates urgency without words.

#### Priority Issues

**[P1] No help or documentation** — Users who don't understand a feature have no path to understanding. The Gist sync instructions are: "create a gist called budget-data.json and enter the gist id and access token with gist modify rights and save" — one sentence covering a multi-step process. Empty states guide creation but offer no help for existing features.

**Why**: First-time users and casual users will abandon the tool when they hit a wall. The Settings modal is the most likely abandonment point.

**Fix**: Add inline help tooltips (question-mark icons) next to complex fields. Rewrite Gist instructions as a step list. Consider a `/help` modal or an FAQ section in Settings.

**Suggested command**: `clarify`

**[P2] No keyboard navigation for item cards** — Item cards use JavaScript click handlers on `.item-clickable` divs but lack `role="button"`, `tabindex="0"`, and keyboard event handlers. Tab key skips past all items without visible focus.

**Why**: Keyboard-only users cannot interact with budget items at all. This is an accessibility failure.

**Fix**: Add `role="button"` and `tabindex="0"` to `.item-clickable` divs. Add keyboard handler for Enter/Space to match click behavior. Add visible `:focus-visible` style distinct from `:hover`.

**Suggested command**: `harden`

**[P2] "Spend" / "Paid" inconsistency** — Budget and Goals items show a "Spend" button. Bills items show a "Paid" button. Both perform the same action (record a deduction), but the label difference signals different semantics.

**Why**: Users learning the app may wonder if "Paid" does something different from "Spend." Predictability principle violation.

**Fix**: Standardize on one label ("Spend" is more universal) or visually group the bill-specific action differently.

**Suggested command**: `clarify`

**[P3] No loading states** — Gist sync operations (save, load) trigger no visible progress indicator. The status bar appears after completion. For slow networks, users see nothing for seconds.

**Why**: Visibility of system status failure. Users may click again, re-save, or navigate away, assuming nothing happened.

**Fix**: Add a loading spinner or dimming overlay during async operations. Disable buttons during sync.

**Suggested command**: `harden`

**[P3] No bulk operations** — Deleting, editing, or resetting multiple items requires individual action on each. Auto Fill exists but only populates based on recurrence — no multi-select, no batch delete.

**Why**: Power users (month-end cleanup, bulk category changes) will find the app tedious. Flexibility and Efficiency deficiency.

**Fix**: Add multi-select mode with batch actions, or swipe-to-delete on mobile.

**Suggested command**: `craft`

#### Persona Red Flags

**Alex (Power User)**: No keyboard shortcuts. No multi-select. No bulk edit. Every item action requires tap → modal → confirm → done. Auto Fill helps but doesn't cover batch deletion or category changes. Alex will feel the friction after 3+ items.

**Jordan (First-Timer)**: Empty states guide creation but Jordan will hit the Settings modal and see an opaque Gist form with no explanation of what a Gist is or why they'd need one. The "Install as App" section is the only clearly explained feature. Jordan may never find the Auto Fill button (⚡ icon is cryptic).

#### Minor Observations

- The ⚡ icon for Auto Fill is not self-explanatory — might be confused with "speed up" or "refresh"
- The auto-fill modal has good UX but no way to see what was auto-filled after dismissal
- Section toggle checkboxes in Settings lack visual indication of their current state (checked = visible, but no preview of what hidden means)
- The delete-spend buttons in edit modals are small red ✕ targets — could be hard to hit on mobile despite 44px touch target pass elsewhere
